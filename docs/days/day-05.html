Day 5

Clamp and Round a Speed Value

Mathf Variables & Data Input
Aim

Use Mathf.Clamp, Mathf.Round, and Mathf.Abs to safely control a floating-point value that represents a player speed. By the end of this task, you should understand how to keep values within limits, round them for readability, and measure how far they are from a target.

Objectives / Pseudocode Breakdown
  • 1. Make a new script called MathfClampAndRound.cs and attach it to an empty GameObject in the scene.
  • 2. Declare and initialise float fields to represent a speed and its allowed range. 
    float currentSpeed = 0f;
float minSpeed = 0f;
float maxSpeed = 10f;
float speedStep = 1.25f; // How much we change speed per key press
  • 3. In Start(), log an initial summary of your speed values to the Console.
    • For example: "Speed start: 0.00 (min 0, max 10)".
  • 4. In Update(), respond to key presses to increase or decrease currentSpeed.
    • Use one key (e.g. up arrow) to add speedStep.
    • Use another key (e.g. down arrow) to subtract speedStep.
    • Use Input.GetKeyDown(KeyCode.UpArrow) and Input.GetKeyDown(KeyCode.DownArrow) so it only changes once per press.
  • 5. After modifying currentSpeed, clamp it between minSpeed and maxSpeed.
    • Use currentSpeed = Mathf.Clamp(currentSpeed, minSpeed, maxSpeed);
  • 6. Round currentSpeed to two decimal places for display.
    • One common pattern is: float rounded = Mathf.Round(currentSpeed * 100f) / 100f;
  • 7. Use Mathf.Abs to calculate how far your speed is from the maximum.
    • For example: float distanceFromMax = Mathf.Abs(maxSpeed - currentSpeed);
  • 8. Log a single summary line each time the speed changes, including the rounded speed and distance from max.
    • Example format: "Speed: 3.75 (distance from max: 6.25)"

The Console shows clamped, rounded speed values and how far they are from the maximum as you tap the arrow keys.

Did I Get It Right?
  • Pressing your chosen increase/decrease keys should adjust the speed value up or down in clear steps.
  • The behaviour should:
    • Never let the speed go below minSpeed or above maxSpeed, no matter how many times you press the keys.
    • Print a single summary line per key press, showing a rounded speed value.
    • Show a sensible distance-from-max number that reaches 0 when you hit the maximum speed.
