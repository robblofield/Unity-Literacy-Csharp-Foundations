Day 2

Toggle Controls

Debugging Input Conditionals
Aim

Explore how to detect when a value changes from one frame to the next. This is an important early skill for avoiding unnecessary logging and reacting only when state actually changes.

Objectives / Pseudocode Breakdown
  • 1. Make a new script called ToggleSwitch.cs and attach it to any GameObject.
  • 2. Create and initialise two booleans: isActive and previousIsActive. 
    bool isActive = false;         // Current state
bool previousIsActive = false; // State from last frame
  • 3. Inside Update(), use the space key to flip isActive.
    • Use Input.GetKeyDown(KeyCode.Space)
    • Remember: if a bool does not equal itself, the new value is the opposite (isActive = !isActive;).
  • 4. Check if the value has changed since last frame (isActive != previousIsActive).
  • 5. Log a message only when the value changes.
    • If true → “Activated”
    • If false → “Deactivated”
  • 6. Update previousIsActive at the end to match the new value.
Hints
  • Hint 1: Don’t worry if it feels like you have a lot of if statements at first. Early scripts often look this way until you learn more compact patterns.
  • Hint 2: A common pattern for detecting change is comparing the current value with a stored value from the previous frame: if (current != previous) { /* something changed */ }
Did I Get It Right?
  • Pressing the spacebar should toggle the value of isActive.
  • You should see one log line each time the value flips:
    • “Activated” when switching to true
    • “Deactivated” when switching back to false
  • You should not see any console spam when the value is not changing.

A look at how your toggle state updates only when the value changes.
