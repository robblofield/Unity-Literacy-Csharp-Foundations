Day 3

Health and Mana Tracker

Variables & Data Debugging Mathf
Aim

Create a simple health and mana tracker using integers and log their values to the Console. By the end of this task, you should understand how to store, change, and safely clamp numeric data in a script.

Objectives / Pseudocode Breakdown
  • 1. Make a new script called HealthManaTracker.cs and attach it to an empty GameObject in the scene.
  • 2. Declare and initialise integer fields for max and current health and mana. 
    int maxHealth = 100;
int currentHealth = 100;
int maxMana = 50;
int currentMana = 50;
  • 3. In Start(), print a summary of your starting values to the Console.
    • Include both current and max values, for example: HP: 100/100 | MP: 50/50.
  • 4. In Update(), respond to a couple of key presses to change these values.
    • One key to apply damage (reduce health) and one to heal (increase health).
    • One key to spend mana and one to restore mana.
    • Use [1] Input.GetKeyDown(KeyCode.X) style checks so each key press only applies once.
  • 5. After changing any value, clamp currentHealth and currentMana so they stay between 0 and their max values.
    • Use [2] Mathf.Clamp to keep values in range.
  • 6. Log the updated summary to the Console whenever a change happens.
    • Use a single formatted line like HP: currentHealth/maxHealth | MP: currentMana/maxMana.
Hints
  • Hint 1: Use int for whole-number values like health and mana. If you find yourself repeating the same summary Debug.Log code, consider putting it in a separate method and calling it after each change.
  • Hint 2: Mathf.Clamp(value, min, max) returns a new value that is forced into the given range. A typical pattern is: currentHealth = Mathf.Clamp(currentHealth, 0, maxHealth);

The Console prints a compact HP/MP summary each time you apply damage, heal, spend, or restore mana.

Did I Get It Right?
  • Pressing your chosen keys should change health and mana values in sensible steps.
  • The Console should show:
    • A starting line like HP: 100/100 | MP: 50/50 when you enter Play mode.
    • One new line each time you press a key that changes health or mana.
    • No values dropping below 0 or going above their max values, no matter how many times you press the keys.
