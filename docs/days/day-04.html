Day 4

Player Stats Summary

Variables & Data Strings Debugging
Aim

Build a simple "character sheet" in code by storing player stats as variables and combining them into a single, readable summary line in the Console. By the end of this task, you should be comfortable using strings to represent grouped information.

Objectives / Pseudocode Breakdown
  • 1. Make a new script called PlayerStatsSummary.cs and attach it to an empty GameObject in the scene.
  • 2. Declare and initialise fields for basic player stats. 
    string playerName = "Nyx";
int level = 1;
int experience = 0;
bool isAlive = true;
  • 3. In Start(), build a single summary string that includes all of these values.
    • Choose a clear format, for example: "Nyx - Lv 1 - XP: 0 - Alive: True".
    • Use either string concatenation (+) or string interpolation ($"...").
  • 4. Log this summary line to the Console using Debug.Log.
  • 5. Still inside Start(), simulate a simple change to your stats.
    • For example: increase level, add some experience, or set isAlive = false;.
  • 6. Build and log a second summary line after the change so you can compare "before" and "after" in the Console.
Hints
  • Hint 1: String interpolation can make long summaries easier to read: $"{playerName} - Lv {level} - XP: {experience} - Alive: {isAlive}". This is often clearer than lots of + "text" concatenations.
  • Hint 2: Try to keep your formatting consistent so it looks like a neat line in the Console. Imagine you are designing a tiny one-line character sheet.

Did I Get It Right?
  • When you enter Play mode, you should see at least two lines printed in the Console.
  • The lines should:
    • Show the same player name in both lines.
    • Show different stats in the second line (for example a higher level or more XP).
    • Use a consistent, readable format across both prints.
